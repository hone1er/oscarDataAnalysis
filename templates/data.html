    Importing Datasets

    We looked on kaggle for relevant datasets and found some very useful files containing movie data, awards data, and cast and crew data for the movies listed. We found these to be incomplete and needed supplemental data to make it useful.

    Scraping the Data

    We created numerous scrapers to fill out our datasets. We scraped the Academy Awards Site and IMDB (using TMDB’s API).

    Merging Data

    We merged our newly scraped data to our existing datasets. This involved some cleaning/standardization of values to ensure everything worked correctly. We then dropped irrelevant data and created a master csv.

    Normalizing our Data

    Our dataset included a lot of movies that didn’t win. We wanted to normalize our testing data to make sure our model predicted things more accurately.

    Correlation Analysis

    We checked the heatmap to see if there were any interesting charts we could play with. We decided to take a deeper look at budget, revenue, vote count, vote average, popularity, and runtime.

    Feature Engineering

    We had a great deal of data for cast and crew, which we wanted to use for our model. We stripped those down, transformed them to numerical values, and ran them.

    Model Training

    We separated our training and test data, scaled them, then ran them.

    Predictions

    We ran our logistical regression, and created a prediction using 2018 data as our test. We were able to accurately determine the winner of the Best Picture award (with one false positive).

    Limitations and Challenges

    • Accurate, if any, budget figures are very hard to find for most foreign movies in general, and all movies prior to ~the mid 1980's
    • "A Day in the Country" (1936) is listed as 1946 on imdb
    • If the page has no image it breaks my scraper, hence the batch jobs of scrapes
    • Some countries no longer exist, should they be used as-is or normalized to their present day entities? (USSR, West Germany, Yugoslavija, Czechoslovakia)
    • Some currencies no longer exist and I don't know if random personal sites I find the convert them are accurate in figures as well as exchange rate for when the movie was (₣, DEM, NLG)
    • Dates and budgets among other data is often in different forms/currencies - it seems IMdB heavily relies upon user input like wikipedia...but without the quality/standards controls.
    • Many releases are around the Cannes film fest (Mid-May) or in October/November right at the end of the window of time for eligibility for the Oscars
    • It is very possible the release date/country data is incorrect since it would seem that often the date shown on the main movie page is very much NOT the earliest or "premiere" date for it.

    Closing Remarks

    We had some difficulty formatting and transforming the data, in particular with Python not recognizing a list of dictionaries as a JSON object (Solution was eval()). Refactoring to split up tasks in your code will leave a much cleaner notebook, though for testing it’s understandable we would go back and forth through a notebook. We trusted our code to working in a way that we expected, but found out later on that it was excluding the results we wanted and we scrambled to get it working properly; always test your code to ensure it is doing not what you’re asking, but what you’re expecting.

    Limitations

    Late in the project the API that the original dataset used changed it's format. This presented an extra challenge when trying to gather and structure the data to match the required shape. To get the model to work the shape must match perfectly and user input needs to grab the needed data and manipulate it into the correct shape into a viable dataframe shape. Another challenge was finding a way to encode the large number of entries in the cast members data. Feature engineering.

    Time was another limitation. Three fold validation on the model took just under ten hours to complete. With only a few weeks to complete the project, every hour counts.
